= tsqlfmt

`tsqlfmt` is a command-line formatter for T-SQL.
It parses SQL with Microsoft's SQL ScriptDOM, then renders formatted output from the syntax tree.

It reads SQL Prompt-style JSON configuration files and targets SQL Prompt-compatible formatting in automated workflows and editor integrations.
It also supports project-specific options under `formatterExtensions`.

[IMPORTANT]
====
First goal: reproduce the style in `default-style.json`.
If there is demand for other styles later, we can expand from there.
====

== Usage

[source,shell]
----
dotnet run --project src/tsqlfmt -- formatSql --authToken ignored --styleName Default < input.sql
----

The formatter reads from standard input and writes formatted SQL to standard output.

== Building

[source,shell]
----
dotnet test tsqlfmt.slnx

dotnet publish src/tsqlfmt/tsqlfmt.fsproj \
  -c Release \
  -r win-x64 \
  --self-contained true \
  -p:PublishSingleFile=true
----

The Windows publish produces a standalone `tsqlfmt.exe` that runs without a separate .NET installation.

== Development

Format F# code with Fantomas:

----
dotnet tool run fantomas .
----

To enable the checked-in pre-commit hook that formats staged `.fs` files and stops the commit when it rewrites them:

[source,shell]
----
git config core.hooksPath .githooks
----

== Benchmarking

To compare `tsqlfmt` and SQL Prompt on the regression fixture corpus, first build both formatters and set the same environment variables used by the parity tests. Keep their values local and do not paste them into logs or shared output.

[source,shell]
----
python benchmarks/regression_suite_benchmark.py \
  --iterations 15 \
  --warmups 3 \
  --json-out artifact/regression-benchmark.json
----

The script benchmarks the `tests/tsqlfmt.Tests/TestData/*.actual.sql` fixtures through each formatter's SQL Prompt-compatible CLI path, prints the slowest files per formatter, and writes aggregate plus per-fixture timings to JSON.
The benchmark output redacts the SQL Prompt executable and auth token values.
